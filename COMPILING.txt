JPEGView source code readme
***************************

To compile JPEGView you need:

- CMake 3.20 or later (https://cmake.org/download/)
- Visual Studio 2022 (v143 toolset), Community or Professional edition
  (JPEGView 32-bit (x86) and 64-bit (x64) versions are both supported)
- vcpkg (https://github.com/microsoft/vcpkg) for the image codec dependencies
  (libjpeg-turbo, libpng[apng], zlib, libwebp, lcms2, libraw, libjxl, libheif,
  libavif).  The static-* triplet is required because JPEGView links the
  static CRT (/MT, /MTd).
- Windows Template Library (WTL), Version 8.1+
  (http://sourceforge.net/projects/wtl/) installed to deps/WTL-sf
  (automatically populated by cloning the submodule:
   git submodule update --init deps/WTL-sf)


One-time vcpkg setup
---------------------

Clone and bootstrap vcpkg somewhere, e.g. D:\vcpkg:

    git clone https://github.com/microsoft/vcpkg.git D:\vcpkg
    D:\vcpkg\bootstrap-vcpkg.bat -disableMetrics

Set VCPKG_ROOT so CMake and the shell can find it:

    setx VCPKG_ROOT D:\vcpkg

Then install the codec dependencies for both architectures (static triplets):

    vcpkg install --triplet x64-windows-static --x-manifest-root=<jpegview> --x-install-root=D:\vcpkg\installed
    vcpkg install --triplet x86-windows-static-jpegview --x-manifest-root=<jpegview> --x-install-root=D:\vcpkg\installed-x86 --overlay-triplets=<jpegview>/vcpkg-triplets

The x86-windows-static-jpegview custom triplet works around an MSVC internal
compiler error in libraw's x86 Debug build.  If your vcpkg install root or VS
install differs, adjust the paths in CMakePresets.json accordingly.


Building with CMake presets
---------------------------

Four presets are defined in CMakePresets.json, one per architecture/config:

    cmake --preset x64-debug
    cmake --build --preset x64-debug

    cmake --preset x64-release
    cmake --build --preset x64-release

    cmake --preset x86-debug
    cmake --build --preset x86-debug

    cmake --preset x86-release
    cmake --build --preset x86-release

The compiled JPEGView.exe and WICLoader.dll, plus config files, are placed
under build/<preset>/out/<config>/ (e.g. build/x64-release/out/Release/).
Codec libraries are statically linked, so no separate DLLs are needed at runtime.


Building the MSI installer
--------------------------

To also build the WiX MSI installer, enable the JPEGVIEW_BUILD_SETUP option.
This requires WiX Toolset v3.11 or later (https://wixtoolset.org/releases/)
with candle.exe and light.exe available on PATH (or the WIX env var set).

    cmake --preset x64-release -DJPEGVIEW_BUILD_SETUP=ON
    cmake --build --preset x64-release --target JPEGView.Setup

The resulting .msi is written to build/x64-release/msi/.


Please note:
- 64-bit Windows 7 or later is needed to run the 64-bit version.
- Image codec libraries are resolved through vcpkg (static linkage).


NOTE: Visual Studio 2022 is the current active JPEGView development environment.


Legacy notes
************

The old VS2017/VS2019 solution, .vcxproj files, committed prebuilt third-party
libraries, and the dependency build scripts have all been removed in favor of
vcpkg.  If you still need them, clone a tag before the vcpkg migration.
